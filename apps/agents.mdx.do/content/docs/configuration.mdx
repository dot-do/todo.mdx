# Configuration

Learn how to configure agents.mdx for your project.

## Agent Configuration

Agents are configured using frontmatter in MDX files:

```mdx
---
name: my-agent
description: Description of what this agent does
version: 1.0.0
context:
  - type: codebase
    path: ./src
  - type: docs
    url: https://docs.example.com
---
```

## Context Sources

### Codebase

```mdx
<Agent.Context>
  <Source type="codebase" path="./src" include="**/*.ts" />
</Agent.Context>
```

### Documentation

```mdx
<Agent.Context>
  <Source type="docs" url="https://docs.example.com" />
</Agent.Context>
```

### API Endpoints

```mdx
<Agent.Context>
  <Source type="api" url="https://api.example.com/data" />
</Agent.Context>
```

## Workflow Configuration

Define multi-step workflows with conditions and branching:

```mdx
<Agent.Workflow>
  <Step name="check" condition={hasChanges}>
    Check for code changes
  </Step>
  <Step name="test" depends="check">
    Run tests
  </Step>
  <Step name="deploy" depends="test" when={testsPass}>
    Deploy to production
  </Step>
</Agent.Workflow>
```

## Environment Variables

Configure agents using environment variables:

```bash
# .env.local
AGENT_API_KEY=your-api-key
AGENT_WEBHOOK_URL=https://example.com/webhook
```

Access in your agent:

```typescript
const apiKey = process.env.AGENT_API_KEY
```

## TypeScript Configuration

Add type definitions for better IDE support:

```typescript
// agents.config.ts
import type { AgentConfig } from 'agents.mdx'

export const config: AgentConfig = {
  name: 'my-agent',
  context: [
    { type: 'codebase', path: './src' }
  ]
}
```
