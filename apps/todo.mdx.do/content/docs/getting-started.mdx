# Getting Started

This guide will walk you through setting up todo.mdx in your project, from installation to creating your first live TODO.md file.

## Prerequisites

Before you begin, make sure you have:

- **Node.js 20+** - [Download here](https://nodejs.org/)
- **pnpm** (recommended) or npm
  ```bash
  npm install -g pnpm
  ```
- **Git** - For version control and beads sync

## Installation

Install todo.mdx in your project:

```bash
npm install @todo.mdx/core
```

Or use it directly with npx (no installation needed):

```bash
npx todo.mdx --help
```

## Quick Start

### 1. Initialize beads

todo.mdx works with [beads](https://github.com/dot-do/beads), a git-based issue tracker. Initialize it in your project:

```bash
bd init
```

This creates a `.beads/` directory to store your issues locally.

### 2. Create Some Issues

Create a few issues to work with:

```bash
bd create --title="Set up project structure" --priority=1
bd create --title="Write documentation" --priority=2
bd create --title="Add tests" --priority=2
```

### 3. Create Your First TODO.mdx

Create a `TODO.mdx` file in your project root:

```mdx
---
title: TODO
beads: true
---

# {title}

<Stats />

## Ready to Work

<Issues.Ready limit={5} />

## In Progress

<Issues.Status status="in_progress" />

## Completed

<Issues.Status status="closed" />
```

### 4. Compile to Markdown

Convert your TODO.mdx to TODO.md:

```bash
npx todo-mdx compile TODO.mdx
```

This generates a `TODO.md` file with live data from your issues:

```markdown
# TODO

12 open · 3 in progress · 5 closed · 68% complete

## Ready to Work
- [ ] **todo-abc** [P1]: Set up project structure
- [ ] **todo-def** [P2]: Write documentation
- [ ] **todo-ghi** [P2]: Add tests

## In Progress
- [-] **todo-jkl** [P1]: Configure CI/CD

## Completed
- [x] **todo-mno** [P1]: Initialize repository
```

## Working with Issues

### Viewing Issues

List all issues:

```bash
bd list
```

Show issues ready to work (no blockers):

```bash
bd ready
```

Show detailed information about an issue:

```bash
bd show todo-abc
```

### Updating Issues

Start working on an issue:

```bash
bd update todo-abc --status=in_progress
```

Complete an issue:

```bash
bd close todo-abc
```

### Dependencies

Add dependencies between issues:

```bash
# todo-def is blocked by todo-abc
bd dep todo-def --depends-on=todo-abc
```

## Setting Up GitHub Sync

### 1. Install the GitHub App

Visit [todo.mdx.do](https://todo.mdx.do) and install the GitHub App on your repository. This enables two-way sync between beads and GitHub Issues.

### 2. Sync Your Issues

Push your local beads issues to GitHub:

```bash
bd sync
```

This will:
- Create GitHub Issues for local beads issues
- Pull GitHub Issues into local beads
- Keep both in sync

### 3. Automatic Updates

Once the GitHub App is installed, changes sync automatically:

- Creating a GitHub Issue creates a beads issue
- Updating an issue status syncs in both directions
- Closing issues syncs automatically

## Auto-Compile with Git Hooks

Keep your TODO.md automatically updated by adding a pre-commit hook:

Create `.git/hooks/pre-commit`:

```bash
#!/bin/sh

# Compile TODO.mdx before each commit
npx todo-mdx compile TODO.mdx

# Stage the updated TODO.md
git add TODO.md
```

Make it executable:

```bash
chmod +x .git/hooks/pre-commit
```

Now `TODO.md` updates automatically whenever you commit!

## Using with AI Agents

### Claude Code

Add live context to your `CLAUDE.md`:

```markdown
## Current Sprint

See TODO.md for current priorities. Check `bd ready` for unblocked tasks.

Priority tasks:
- Review TODO.md before starting work
- Update issue status with `bd update <id> --status=in_progress`
- Close issues with `bd close <id>` when complete
```

Compile CLAUDE.md before starting work:

```bash
npx todo-mdx compile CLAUDE.md
```

### Cursor

Update `.cursorrules` to reference TODO.md:

```markdown
# Workflow

1. Check TODO.md for current sprint priorities
2. Pick an issue from "Ready to Work" section
3. Update status: `bd update <id> --status=in_progress`
4. When done: `bd close <id>`
5. Recompile: `npx todo-mdx compile TODO.mdx`
```

## Configuration

### Frontmatter Options

Customize your TODO.mdx with frontmatter:

```mdx
---
title: Project TODO
beads: true
filePattern: "[id]-[title].md"
---
```

**Options:**
- `title` - Used in `{title}` placeholders
- `beads` - Enable beads integration (default: true)
- `filePattern` - Pattern for .todo/*.md files

### Config File

Create `todo.mdx.config.ts` for global settings:

```typescript
export default {
  input: ['TODO.mdx', 'ROADMAP.mdx'],
  output: './',

  tracker: {
    type: 'beads',
    path: '.beads',
  },

  defaults: {
    limit: 10,
  },
}
```

## Troubleshooting

### "beads not initialized"

Run `bd init` in your project root.

### "No issues found"

Create issues with `bd create --title="Your task"` or sync from GitHub with `bd sync`.

### Components not rendering

Ensure your TODO.mdx has proper frontmatter and the file is valid MDX syntax.

### GitHub sync not working

1. Check that the GitHub App is installed on your repository
2. Verify you have the correct permissions
3. Try running `bd sync` manually
