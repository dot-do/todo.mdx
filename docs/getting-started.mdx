# Getting Started

todo.mdx compiles MDX templates with live data into plain Markdown. Define your TODO once, render everywhere.

## Installation

```bash
npm install todo.mdx
# or
pnpm add todo.mdx
# or
bun add todo.mdx
```

## Quick Start

### 1. Initialize

```bash
npx todo.mdx init
```

This creates:
- `TODO.mdx` — Your template file
- `.todo/` — Directory for individual issue files

### 2. Write Your Template

```mdx
---
title: Project TODO
beads: true
---

# {frontmatter.title}

<Stats />

## Ready to Work

<Issues.Ready limit={5} />

## In Progress

<Issues.InProgress />

## Blocked

<Issues.Blocked />
```

### 3. Compile

```bash
npx todo.mdx
```

Outputs `TODO.md`:

```markdown
# Project TODO

12 open · 3 closed · 80% complete

## Ready to Work

- [ ] **todo-4fh** [P1]: Create cli.mdx package
- [ ] **todo-8mg** [P1]: Export Payload via Workers RPC
- [ ] **todo-vm0** [P1]: Workflow MDX parser

## In Progress

- [ ] **todo-abc** [P1]: Implement authentication flow

## Blocked

- [ ] **todo-xyz** [P1]: Deploy to production
  - Blocked by: todo-abc (Implement authentication)
```

## Components

### Issues

```mdx
<Issues.Open />        <!-- All open issues -->
<Issues.Closed />      <!-- Completed issues -->
<Issues.Ready />       <!-- No blockers -->
<Issues.Blocked />     <!-- Has blockers -->
<Issues.InProgress />  <!-- Currently being worked -->
```

**Props:**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `limit` | number | ∞ | Max issues to show |
| `priority` | 0-4 | all | Filter by priority |
| `labels` | string[] | all | Filter by labels |
| `assignee` | string | all | Filter by assignee |

### Stats

```mdx
<Stats />              <!-- "12 open · 3 closed · 80% complete" -->
<Stats.Progress />     <!-- Progress bar -->
<Stats.Summary />      <!-- Detailed breakdown -->
```

### Roadmap

```mdx
<Roadmap />                   <!-- All milestones -->
<Milestone id="v1.0" />       <!-- Single milestone -->
<Milestone.Progress id="v1.0" />  <!-- Progress bar -->
```

## Data Sources

todo.mdx pulls data from multiple sources:

### beads (Local)

Local issue tracking in `.beads/` directory:

```bash
# Enable in frontmatter
---
beads: true
---
```

Uses `bd` CLI commands under the hood.

### GitHub Issues

Sync with GitHub via the todo.mdx GitHub App:

```bash
# Enable in frontmatter
---
github:
  repo: owner/repo
  sync: true
---
```

### todo.mdx API

Remote access via REST or MCP:

```bash
# Enable in frontmatter
---
api:
  url: https://todo.mdx.do
  token: $TODO_MDX_TOKEN
---
```

## File Generation

Generate individual `.todo/*.md` files from issues:

```bash
npx todo.mdx --generate
```

Creates files like:
```
.todo/
├── todo-4fh-create-cli-mdx-package.md
├── todo-8mg-export-payload-via-workers-rpc.md
└── todo-vm0-workflow-mdx-parser.md
```

### File Pattern

Customize the naming pattern:

```mdx
---
filePattern: "[id]-[title].md"
---
```

Available tokens:
- `[id]` — Issue ID
- `[title]` — Slugified title
- `[priority]` — Priority level
- `[type]` — Issue type

## Configuration

### Frontmatter Options

```mdx
---
# Basic
title: My Project TODO

# Data sources
beads: true
github:
  repo: owner/repo
  sync: true

# File generation
filePattern: "[id]-[title].md"
outputs:
  - TODO.md
  - .todo/*.md

# Rendering
showPriority: true
showLabels: true
showAssignee: true
---
```

### Config File

Create `todo.mdx.config.ts`:

```typescript
import { defineConfig } from 'todo.mdx'

export default defineConfig({
  input: 'TODO.mdx',
  outputs: ['TODO.md', '.todo/*.md'],
  beads: true,
  github: {
    repo: 'owner/repo',
    sync: true
  }
})
```

## CLI Commands

```bash
# Compile TODO.mdx to TODO.md
npx todo.mdx

# Watch mode
npx todo.mdx --watch

# Generate .todo/*.md files
npx todo.mdx --generate

# Start MCP server
npx todo.mdx --mcp

# Sync with GitHub
npx todo.mdx --sync
```

## MCP Server

Expose todo.mdx to AI assistants:

```bash
npx todo.mdx --mcp
```

Add to Claude Desktop config:

```json
{
  "mcpServers": {
    "todo.mdx": {
      "command": "npx",
      "args": ["todo.mdx", "--mcp"]
    }
  }
}
```

**Available tools:**
- `listIssues` — Query issues with filters
- `createIssue` — Create new issue
- `updateIssue` — Modify issue
- `closeIssue` — Mark complete
- `search` — Full-text search
